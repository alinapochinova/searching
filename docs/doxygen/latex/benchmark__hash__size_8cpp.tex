\doxysubsection{Detailed Description}
Бенчмарк зависимости коллизий и времени поиска от размера хэш-\/таблицы. 

Фиксируется массив из 500 000 элементов (2000 уникальных ключей). Размер хэш-\/таблицы варьируется от 101 до 500009 (простые числа). Для каждого размера измеряется количество коллизий и время поиска.

Последовательно тестирует хэш-\/таблицы разных размеров:\+
\begin{DoxyItemize}
\item Генерирует фиксированный массив данных (один раз).
\item Для каждого размера таблицы строит новую хэш-\/таблицу, вставляет все элементы, замеряет коллизии и время поиска.

\end{DoxyItemize}

\begin{DoxyNote}{Note}
Ожидаемый результат:\+
\begin{DoxyItemize}
\item При малом размере таблицы (101) – много коллизий, время поиска большое.
\item При оптимальном размере (\texorpdfstring{$\sim$}{\string~}2 \texorpdfstring{$\ast$}{*} UNIQUE\+\_\+\+KEYS\+\_\+\+COUNT = 4000) – коллизий мало.
\item При дальнейшем увеличении размера – коллизии стремятся к 0, время поиска стабилизируется, но растёт расход памяти. 
\end{DoxyItemize}
\end{DoxyNote}
